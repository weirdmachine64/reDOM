reDOM is a Burp Suite extension that renders JavaScript-heavy pages using Chrome DevTools Protocol to capture the final DOM state after JavaScript execution.

Features

  • Captures fully-rendered DOM after JavaScript execution
  • Analyzes Single Page Applications (SPAs) built with React, Vue.js, Angular, etc. that Burp can't handle natively
  • Integrates as a custom response tab in Burp Repeater
  • Auto-render option for automatic DOM capture
  • Configurable Chrome connection and rendering parameters

Requirements

  • Burp Suite Professional or Community Edition
  • Chrome/Chromium browser

Setup

  1. Start Chrome with remote debugging enabled: 
    google-chrome --remote-debugging-port=9222 --headless
    or 
    chromium -proxy-server=localhost:8080 --remote-debugging-port=9222 --user-data-dir=/tmp/redom --ignore-certificate-errors
  2. In Burp Suite, navigate to the reDOM Settings tab
  3. Click "Connect to Chrome"
  4. The extension will spawn a minimized browser window for rendering operations
    • Important: Do not close this window - it's used for all rendering operations

Usage

  1. Send a request to Burp Repeater
  2. Switch to the "DOM Render" tab in the response area
  3. Either:
    • Click "Render in Browser" to manually capture DOM content
    • Enable "Auto render" to automatically render when the tab opens

Configuration

The extension provides the following configurable settings:

  • Chrome Host/Port: Connection details for Chrome DevTools Protocol (default: localhost:9222)
  • CDP Command Timeout: Maximum time to wait for WebSocket commands in seconds (default: 30)
  • Page Load Timeout: Maximum time to wait for page load completion in seconds (default: 30)
  • Render Delay: Additional wait time after page load before capturing DOM in milliseconds (default: 1000)
  • Auto Render: Automatically render pages when the DOM Render tab opens
  • Minimized Window: Start the Chrome rendering window minimized

Use Cases

  • Analyzing React, Vue.js, Angular, and other SPA applications
  • Testing JavaScript-heavy applications where traditional Burp responses show incomplete HTML
  • Capturing dynamically generated content after client-side rendering
  • Understanding the final DOM state for security testing
